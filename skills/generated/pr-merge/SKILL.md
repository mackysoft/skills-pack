# pr-merge

## 目的
現在作業または指定 PR を、安全にマージする。

対象 PR を確定し、必要な更新、push、Ready 化、CI 待機、マージ、リモートブランチ後始末まで進める。

## フロー

### Phase 1: 対象 PR を確定する
- `gh auth status` を確認する。
- PR 番号、URL、ブランチが指定されている場合は、その PR を使う。
- 指定がない場合は、現在ブランチの open PR を探す。
- 現在ブランチに PR がなく、現在作業から PR を作れる場合は `$pr-create` を使う。
- 複数候補がある場合は停止し、対象を確認する。

### Phase 2: PR を更新可能な状態にする
- 未コミット差分または未 push コミットがある場合は `$push` を使う。
- PR が behind の場合は `$sync-latest` で base branch を取り込み、`$push` で反映する。
- 同期で衝突した場合はマージへ進まない。
- 既存 PR の本文は、マージのためだけには更新しない。

### Phase 3: Ready にする
Draft PR の場合は Ready にする。
Ready 化できない理由がある場合は停止する。

### Phase 4: チェックを待つ
必須チェックがある場合は完了まで待つ。
必須チェックが検出できない場合は短く待って、表示されたチェックを待つ。

失敗、キャンセル、タイムアウトしたチェックがある場合は停止する。
チェック待機後に最新の head SHA を取得する。

### Phase 5: マージする
`gh pr merge <number> --merge --match-head-commit <headRefOid>` でマージする。
`--auto` は使わない。
マージ本体では `--delete-branch` を使わない。

保護ルール起因で失敗した場合だけ、`--admin` 再試行の可否を確認する。

### Phase 6: リモートブランチを後始末する
リポジトリの自動ブランチ削除設定と、現在のリモートブランチ存在有無を確認する。
自動削除が有効な場合は反映を短く待つ。
待っても残っている場合だけ `git push origin --delete <headRefName>` を実行する。

### Phase 7: 状態を残す
PR URL、更新に使ったスキル、チェック結果、マージ結果、リモートブランチ削除結果、停止理由を残す。
